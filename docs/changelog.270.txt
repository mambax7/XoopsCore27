2.7.0 RC1                    2026-04
===================================
Version and platform:
- renamed version from 2.5.12 to 2.7.0 RC1 (mamba)
- raise minimum PHP version to 8.2 by @mambax7
  - update CI matrix to PHP 8.2/8.3/8.4/8.5, PHPUnit 11
  - update SonarCloud and Qodana analysis configs to PHP 8.2
  - update installer PHP version check to 8.2.0
  - consolidate session handlers into single session.php (remove session74.php)
  - remove dead PHP < 7.3 and < 8.0 code branches
- fix PHP 8.4 deprecated implicit nullable parameters across core (mamba)
- add PHP 8.5 to CI matrix as forward-compatibility check (mamba)
- drop external xoops/base-requires25 metapackage; all libraries are now listed directly in composer.dist.json (mamba)
- rename Composer meta-package from xoopscore25/libraries to xoopscore27/libraries — downstream consumers referencing the old name in their own require blocks must update accordingly (mamba)

Security hardening (RC candidate review):
- add CSRF token to module admin AJAX requests by @mambax7 in #1685
- add configurable session cookie SameSite and Secure preferences by @mambax7 in #1501, #1660
- fix SonarCloud blocker XSS vulnerabilities: escape all reflected user data (mamba) in #1662
- add exec() override to Protector dblayertrap and check exec results (mamba) in #1644
- remove eval() from PHP blocks and Protector lifecycle legacy cleanup (mamba) in #1653
- all unserialize() calls now use allowed_classes restriction to prevent object injection (mamba) in #1636, #1638
- validate Protector prefix input instead of silently rewriting (mamba) in #1661
- fix Protector badips file handling, replace die() with exception in validatePrefix (mamba)
- fix instanceof operator precedence in result set guards (mamba) in #1654
- use strict comparison for passwords, pin uname Request to POST (mamba)
- fix AJAX toggle handlers to read IDs from POST, not GET (mamba)
- sanitize errfile path before storing and dispatching to sub-loggers (mamba)
- clamp search limit to prevent unbounded queries, harden test file reads (mamba)
- harden upgrade with scoped queries and duplicate prevention (mamba)
- tighten hash_equals assertions to target actkey comparison specifically (mamba)
- URL scheme whitelist: decode entities before checking scheme, check only scheme portion (mamba)
- JSON_HEX_TAG in <script> blocks to prevent </script> breakout (mamba)
- basename() before filename allowlist to block directory traversal via encoded paths (mamba)
- mb_strlen() for multibyte-aware length validation across form input (mamba)
- fix for XOOPS_COOKIE_DOMAIN (mamba) in #1578 (see: https://xoops.org/modules/newbb/viewtopic.php?topic_id=79632&post_id=367525#forumpost367525)
- improve validation of $_SERVER['HTTP_HOST'] (mamba) in #1578
- update htdocs/include/xoopssetcookie.php (mamba) in #1578
- fix custom block c_type lowercase regression from Request::getCmd() (mamba) in #1657
- fix Elvis operator dropping zero values in Request::getXxx() fallbacks (mamba) in #1650

Admin UI and themes:

- clean-room system menu administration rewrite by @mambax7 in #1666
- fix system menu table creation and xbootstrap5 dropdown styling (mamba) in #1676

- feature: change module display order for all modules in #1677
- add renderFooterInfo and displayFooterInfo methods to moduleadmin in #1679
- fix for module admin required field asterisk in #1673
- fix breadcrumbs and xoAdminIcons consistency across system admin pages (mamba)
- add menu icon support for custom module submenus (mamba)

Refactors and code quality:

- consolidate convention tests into single method (SonarCloud duplication gate) (mamba)
- remove legacy @package/@subpackage/@category tags per PSR-12 (mamba)
- database query modernization: deprecate queryF() — use exec() for writes/DDL and query() for SELECTs; deprecate quoteString() — use quote() (mamba) in #1646
- criteria cleanup: IN clause uses array passing instead of string concat (mamba) in #1572
- mailer cleanup (mamba) in #1574

- MyISAM compensating restore pattern for avatar delete — capture UIDs, restore on failure (mamba) in #1651
- use getVar('n') for fields with intentional HTML to prevent double-escaping (mamba) in #1665
